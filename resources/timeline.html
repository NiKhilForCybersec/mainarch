Why This Order?

Step Why This Order What Breaks If Wrong Order
1. Sentinel workspace first All connectors need a destination Can't enable connectors without workspace
2. M365 Defender connector early Gets XDR data flowing to Sentinel immediately (FREE) Missing correlation data, paying for data you get free
3. MDE before MDI Endpoint context enriches identity alerts MDI alerts lack device context for investigation
4. MDI before MDCA User risk scores inform MDCA policies MDCA can't leverage identity risk for session controls
5. All XDR products before detection engineering Need data flowing to build detections Writing rules against empty tables
6. DLP last (parallel track) Least dependent on other products N/A — can run independently

════════════════════════════════════════════════════════════════════════════════

• Security!*[System[(EventID=4648)]]    -- Explicit credentials
• Security!*[System[(EventID=4672)]]    -- Special privileges

• Security!*[System[(EventID=4698)]]    -- Scheduled task created

• Security!*[System[(EventID=5156)]]    -- Firewall allow (sampled)

• EventID 4634 (Logoff) — Low security value
• EventID 5157 (Firewall block) — Too verbose

- Syslog from Linux

- Custom logs

• August 2024: MMA officially deprecated
• August 2025: MMA end of support
• December 2025: MMA no longer receives updates

WHAT WE DID:
════════════════════════════════════════════════════════════════════════════════
1. INVENTORY
   - Identified all servers with MMA installed
   - ~150 Windows servers, ~80 Linux servers

2. DEPLOY AMA
   - Created Data Collection Rules (DCRs) in Azure
   - DCRs define: which logs to collect, where to send them
   - Deployed AMA via Azure Policy (auto-install on new VMs)
   - Manual install on on-prem servers via Arc

3. CONFIGURE DCRs
   - Windows Security Events DCR:
     └─ XPath queries to filter specific Event IDs
     └─ Destination: Log Analytics workspace
   
   - Syslog DCR:
     └─ Facilities: auth, authpriv, syslog, daemon, kern
     └─ Min severity: Warning
   
   - CEF DCR:
     └─ Forwarder VM with rsyslog → AMA → CommonSecurityLog

4. VALIDATE
   - Confirmed data flowing to same tables
   - Updated any KQL queries referencing agent fields

5. REMOVE MMA
   - Uninstalled MMA after parallel run
   - Removed legacy solutions from workspace
