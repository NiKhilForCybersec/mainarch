Infrastructure: Azure Migration

Foundation Track Status: In Progress

Infrastructure migration started first alongside Identity as the cloud foundation ‚Äî you can't migrate security tools until cloud infrastructure is in place. Azure landing zone, network architecture, and connectivity are established. Workload migration is ongoing with data center decommission targeted for Q4 2025.

Why Azure Migration Is Happening

Context: RevFlow was founded in 2015 with traditional on-premises infrastructure‚Äîa co-located data center in Toronto with VMware virtualization, ~200 VMs, on-prem Active Directory. This made sense at the time for a B2B SaaS startup.

Migration Drivers:

  • Siemens acquisition (2022): Azure-first mandate for all subsidiaries
  • Data center lease expiring Q4 2025: Migrate to Azure or renew 5-year commitment
  • Scalability needs: On-prem infrastructure hitting capacity limits
  • Remote work reality: VPN + on-prem AD model showing strain post-COVID
  • Operational overhead: Hardware maintenance, rack space, cooling, power

The security migration (RSA ‚Üí Sentinel, ESET ‚Üí MDE, Symantec ‚Üí Purview) is part of this larger Azure transformation. Security tools are moving to cloud-native platforms as the underlying infrastructure moves to Azure.

What's Still On-Premises

The security migration is complete, but workload migration is still in progress:

  • Active Directory: Hybrid (synced to Entra ID, will decommission after full migration)
  • Production application servers: Some still in data center, being migrated
  • PostgreSQL databases: Customer data still in data center, migrating to Azure PostgreSQL
  • Palo Alto firewalls: Still active for data center egress
  • Symantec Network DLP: Active until Azure network architecture complete

Target: Full data center decommission by Q4 2025 (lease expiration)

Industry Best Practices

Cloud Infrastructure Best Practices

Microsoft Cloud Adoption Framework (CAF):

  • Landing Zone First: Establish governance, networking, identity before workloads
  • Hub-Spoke Network: Centralized connectivity and security controls
  • Policy as Code: Azure Policy for compliance guardrails
  • Identity Foundation: Entra ID as single identity plane

CIS Azure Benchmark:

  • Enable Defender for Cloud on all subscriptions
  • Encrypt data at rest and in transit
  • Use NSGs and Azure Firewall for network segmentation
  • Enable diagnostic logging to Log Analytics
