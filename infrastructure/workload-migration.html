Workload Migration

Infrastructure Track

Phased migration of on-premises workloads to Azure. Each workload is assessed for the appropriate migration strategy.

Migration Strategies

StrategyDescriptionWhen to Use
Rehost (Lift & Shift)Move VM as-is to AzureQuick migration, minimal changes
RefactorMinor modifications for cloudUse PaaS where possible
RearchitectSignificant redesignPerformance/scale requirements
RebuildBuild new from scratchLegacy systems, technical debt
ReplaceSwitch to SaaSCommodity workloads

Migration Order

WaveWorkloadStrategyTarget
1Dev/Test environmentsRehostAzure VMs

3RevFlow API (non-prod)RefactorApp Services

5RevFlow API (prod)RefactorApp Services

Database Migration

Azure Database Migration Service
# Create DMS instance
az dms create \
  --resource-group "rg-migration" \
  --name "dms-revflow" \
  --location "eastus" \
  --sku-name "Premium_4vCores"

# Migration workflow:
1. Assess source SQL Server with Data Migration Assistant
2. Create Azure SQL MI target
3. Configure DMS project
4. Run online migration (minimal downtime)
5. Cutover during maintenance window

Validation Checklist

Post-Migration Validation
  • ☐ Application health checks passing
  • ☐ Database connectivity verified
  • ☐ Performance baseline established
  • ☐ Monitoring configured (Application Insights)
  • ☐ Backup policies applied
  • ☐ Defender for Cloud recommendations addressed
  • ☐ DNS cutover complete
