Azure Firewall Rules

Azure CLI
# Application rules - allow outbound HTTPS
az network firewall application-rule create \
  --firewall-name "fw-hub-prod" \
  --resource-group "rg-platform" \
  --collection-name "Allow-Web" \
  --name "AllowMicrosoft" \
  --protocols Https=443 \
  --source-addresses "10.0.0.0/8" \
  --target-fqdns "*.microsoft.com" "*.azure.com" "*.windows.net" \
  --action Allow \
  --priority 100

# Network rules - allow DNS
az network firewall network-rule create \
  --firewall-name "fw-hub-prod" \
  --resource-group "rg-platform" \
  --collection-name "Allow-DNS" \
  --name "AllowDNS" \
  --protocols UDP \
  --source-addresses "10.0.0.0/8" \
  --destination-addresses "168.63.129.16" \
  --destination-ports 53 \
  --action Allow \
  --priority 100

Industry Best Practices

Landing Zone Best Practices

Microsoft Cloud Adoption Framework:

  • Hub-Spoke Topology: Centralized connectivity and security controls
  • Azure Firewall: Centralized egress filtering and threat intelligence
  • Management Group Hierarchy: Governance at scale with inherited policies
  • Policy as Guardrails: Prevent non-compliant deployments before they happen

CIS Azure Foundations Benchmark:

  • Enable Defender for Cloud (Standard tier) on all subscriptions
  • Require secure transfer for storage accounts
  • Enable Network Watcher for all regions
  • Configure diagnostic settings for all resources
