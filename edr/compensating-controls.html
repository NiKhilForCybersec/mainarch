MDE Agent Gaps & Compensating Controls

EDR Track Comprehensive Coverage Strategy

Not everything can run an MDE agent. This page documents all asset categories where MDE cannot be deployed and the compensating controls we implement to maintain security visibility. The goal isn't 100% MDE coverage â€” it's 100% visibility through the right tool for each asset class.

Key Principle: Coverage â‰  Agent Deployment

The goal is security visibility, not agent deployment percentage.

For systems where MDE can't be installed, we implement layered compensating controls:

  • Network-level visibility: Defender for IoT, NDR, firewall telemetry
  • Log ingestion: Syslog, CEF, API integrations to Sentinel
  • Network segmentation: Isolate unmanaged devices
  • Access controls: Conditional Access, PAM, MFA

Compensating Control Types

Defender for IoT

Passive network monitoring for OT/IoT devices. No agent required â€” analyzes network traffic for anomalies and threats.

Syslog â†’ Sentinel

Log forwarding via Syslog, CEF, or LEEF. Brings device logs into Sentinel for detection and correlation.

Network Segmentation

VLANs, firewall rules, microsegmentation. Isolates unmanaged devices and limits blast radius.

Defender for Cloud Apps

CASB for SaaS visibility. OAuth app monitoring, session controls, data loss prevention.

Defender for Cloud

Cloud security posture management. Protects Azure PaaS, serverless, containers without agents.

PAM / Access Control

CyberArk, PIM, session recording. Controls privileged access to sensitive systems.

Alternative EDR

CrowdStrike Falcon, Carbon Black, Wazuh. For platforms MDE doesn't support (AIX, Solaris, etc.).

Conditional Access

Entra ID Conditional Access. Block non-compliant devices, enforce MFA, limit app access.
