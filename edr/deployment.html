Intune Deployment (Recommended)

Step 1: Configure MDE Settings

Intune Portal Steps
1. Microsoft Intune admin center → Endpoint security
2. Microsoft Defender for Endpoint → Open connector
3. Enable connection between Intune and MDE
4. Configure settings:
   ☑ Connect Windows devices version 10.0.15063 and above
   ☑ Connect Windows devices version 10.0.18362 (Server 2019)
   ☑ Block unsupported OS versions
   ☑ Allow Microsoft Defender for Endpoint to enforce Endpoint Security Configurations

Step 2: Create Configuration Profile

Intune Configuration
Devices → Configuration profiles → Create profile

Platform: Windows 10 and later
Profile type: Templates → Microsoft Defender for Endpoint (Desktop)

Configuration settings:
- Microsoft Defender for Endpoint client configuration package: Onboarding
- Sample sharing: All
- Expedite telemetry reporting frequency: Enable
- Enable/Disable: Enable

Assignments:
- Deploy to "All Devices" or phased groups

Step 3: Verify Onboarding

PowerShell - On Endpoint
# Check MDE onboarding status
$regPath = "HKLM:\SOFTWARE\Microsoft\Windows Advanced Threat Protection\Status"
Get-ItemProperty -Path $regPath | Select-Object OnboardingState

# OnboardingState = 1 means successfully onboarded

# Test detection (EICAR)
Invoke-WebRequest -Uri "https://www.eicar.org/download/eicar.com.txt" -OutFile "$env:TEMP\eicar.txt"
# Should be blocked by MDE

SCCM Deployment

SCCM Task Sequence
1. Download onboarding package from security.microsoft.com
   Settings → Endpoints → Onboarding → Download package

2. Create SCCM Application:
   - Content: WindowsDefenderATPOnboardingScript.cmd
   - Installation program: WindowsDefenderATPOnboardingScript.cmd
   - Detection method: Registry key exists
     HKLM\SOFTWARE\Microsoft\Windows Advanced Threat Protection\Status\OnboardingState = 1

3. Deploy to collection with phased rollout:
   - Week 1: IT Department (50 devices)
   - Week 2: Engineering (150 devices)
   - Week 3: Sales + Marketing (200 devices)
   - Week 4: All remaining

Onboarding Validation Query

KQL - Advanced Hunting
// Check device onboarding status
DeviceInfo
| where Timestamp > ago(24h)
| summarize arg_max(Timestamp, *) by DeviceId
| project 
    DeviceName,
    OSPlatform,
    OSVersion,
    OnboardingStatus,
    SensorHealthState,
    LastSeen = Timestamp
| where OnboardingStatus == "Onboarded"
| summarize 
    TotalOnboarded = count(),
    HealthyDevices = countif(SensorHealthState == "Active"),
    InactiveDevices = countif(SensorHealthState != "Active")
