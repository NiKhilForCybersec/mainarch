Coexistence Strategy

EDR Track

Configuring ESET and MDE to run simultaneously without conflicts. Mutual exclusions prevent scanning loops and performance degradation.

Exclusion Architecture

Both products must exclude the other's processes and folders. This is a bidirectional configuration:

ESET Protect
Exclude MDE processes
Exclude MDE folders
Microsoft Defender
Exclude ESET processes
Exclude ESET folders

Intune Configuration

Intune PowerShell
# Create MDE exclusions policy via Intune
# Endpoint Security → Antivirus → Create Policy

$exclusions = @{
    "ProcessExclusions" = @(
        "C:\Program Files\ESET\ESET Security\ekrn.exe",
        "C:\Program Files\ESET\ESET Security\egui.exe"
    )
    "PathExclusions" = @(
        "C:\Program Files\ESET\",
        "C:\ProgramData\ESET\"
    )
}

# Apply via Intune Antivirus policy

Performance Impact

Expected Overhead

Running two EDR agents increases resource usage:

  • RAM: Additional 500MB - 1GB per endpoint
  • CPU: 5-10% increase during scans
  • Disk I/O: Minimal with proper exclusions

This is acceptable for the 8-10 month coexistence period. Monitor and add exclusions if specific applications struggle.
