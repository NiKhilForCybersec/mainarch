Built-In SITs

Purview includes 300+ built-in Sensitive Information Types. Use these before creating custom SITs. Common ones: Credit Card Number, U.S. Social Security Number, Bank Account Number, SWIFT Code.

P3: Employee PII

PowerShell - DLP Policy
# Create DLP Policy for Employee PII
New-DlpCompliancePolicy -Name "Employee PII Protection" `
    -Mode "TestWithNotifications" `  # Audit mode first
    -ExchangeLocation All `
    -SharePointLocation All `
    -OneDriveLocation All `
    -TeamsLocation All

# Add rule to policy
New-DlpComplianceRule -Name "Block SSN sharing" `
    -Policy "Employee PII Protection" `
    -ContentContainsSensitiveInformation @{
        Name = "U.S. Social Security Number (SSN)"
        MinCount = 1
        MaxConfidence = 100
        MinConfidence = 75
    } `
    -BlockAccess $true `
    -NotifyUser "SiteAdmin,LastModifier,Owner" `
    -NotifyEmailCustomText "This document contains SSN data and cannot be shared externally."

Endpoint DLP Configuration

For endpoint protection (USB, printing, clipboard), Purview Endpoint DLP requires MDE onboarding:

Endpoint DLP Policy
# Enable Endpoint DLP for devices
Set-PolicyConfig -EndpointDlpEnabled $true

# Create Endpoint DLP rule
New-DlpComplianceRule -Name "Block USB for Customer Data" `
    -Policy "Customer Data Protection" `
    -ContentContainsSensitiveInformation @{
        Name = "RevFlow Customer Data"
        MinCount = 1
    } `
    -EndpointDlpRestrictions @{
        # Actions and their modes
        CopyToRemovableMedia = "Block"
        CopyToNetworkShare = "Warn"
        CopyToClipboard = "Audit"
        Print = "Warn"
        UploadToCloudService = "Block"
    }
