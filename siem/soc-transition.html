Alert Deduplication

During parallel operations, both systems will generate alerts for the same events. Procedure:

  1. Acknowledge alert in both consoles
  2. Investigate in the designated primary console
  3. Document findings in Sentinel incident (single source of truth)
  4. Close in both systems with same resolution

Example: Auto-Enrichment Playbook

Logic App Trigger
Trigger: Microsoft Sentinel incident
  ↓
Action: Get incident entities (IPs, Users, Hosts)
  ↓
For each IP:
  → Query VirusTotal API
  → Query AbuseIPDB
  → Add results to incident comments
  ↓
For each User:
  → Query Entra ID for risk level
  → Query recent sign-in locations
  → Add results to incident comments
  ↓
If high-risk indicators found:
  → Update incident severity to High
  → Send Teams notification to SOC channel

Cutover Checklist

Go-Live Readiness
  • ☐ All critical detection rules migrated and validated
  • ☐ All analysts completed KQL training
  • ☐ All automation playbooks tested
  • ☐ Log source coverage verified (no gaps)
  • ☐ Escalation procedures updated
  • ☐ On-call rotation updated for Sentinel
  • ☐ Stakeholder communication sent
  • ☐ Rollback plan documented
