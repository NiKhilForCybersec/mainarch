SIEM Track Complete Rule Disposition

This catalog documents the disposition of every RSA NetWitness detection rule for RevFlow Analytics — a B2B SaaS Revenue Intelligence platform with SOC 2 Type II, ISO 27001, GDPR, and Siemens corporate compliance requirements. Each rule evaluated using the 6-Step Decision Framework.

Why Migration ≠ Lift-and-Shift

RSA NetWitness did NOT have adequate coverage. It was deployed in 2018 when RevFlow was primarily on-prem with limited cloud/SaaS adoption. The rule set reflected that era — network-centric, endpoint-focused, with minimal cloud coverage.

This migration is a detection modernization, not a translation exercise.

Understanding the Numbers

487 RSA rules evaluated:

  • 312 retired to XDR — Endpoint, identity, email detection now handled by Microsoft Defender with superior fidelity
  • 117 migrated to Sentinel — Network, cloud infrastructure rules rewritten in KQL
  • 58 dropped — Stale IOCs, generic thresholds, single-event noise with no detection value

+230 new Sentinel rules — NOT derived from RSA. Built from scratch to cover gaps RSA never addressed:

  • SaaS application monitoring (Salesforce, Workday, NetSuite, GitHub) — RSA had no connectors
  • Business logic detection (terminated employee correlation, VIP monitoring) — RSA couldn't correlate with HR systems
  • Compliance monitoring (SOC 2 controls, GDPR data access) — RSA had no compliance framework
  • Higher-order XDR correlation — RSA had no XDR to correlate with

Total Sentinel detection rules: 347 — appropriate for B2B SaaS with SOC 2/ISO 27001.

Interview Talking Point

"The RSA migration wasn't lift-and-shift — it was detection modernization. RSA was deployed in 2018 when the company was 70% on-prem. By 2024, we were 90% cloud with crown jewels in Salesforce and Workday. RSA had 17 SaaS rules total; we needed 87. It had zero compliance monitoring; we needed 45 rules for SOC 2. The 230 new rules aren't gaps we discovered — they're coverage for a threat landscape RSA was never designed to address."
