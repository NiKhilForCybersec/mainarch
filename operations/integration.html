Key Integrations

XDR-First Data Flow

Critical: We ingest XDR alerts and incidents, NOT raw endpoint telemetry. Raw Device* tables stay in XDR and are queried via Advanced Hunting. This saves ~$15K/month.

IntegrationData FlowValue
M365 Defender → SentinelAlerts + Incidents only (NOT raw telemetry)Unified incident queue, correlation with identity/network
Purview → SentinelDLP alertsCorrelate data exfiltration with identity events
Entra ID → SentinelSign-in + audit logs (control-plane)Identity changes XDR doesn't capture
Defender for Cloud → SentinelSecurity alerts + recommendationsCloud misconfig visibility

Bi-directional Sync

M365 Defender incidents synced to Sentinel maintain bi-directional status updates:

  • Close incident in Defender → Auto-closes in Sentinel
  • Add comment in Sentinel → Appears in Defender
  • Change severity in either → Syncs to other
Unified SOC Experience

Primary Interface: security.microsoft.com — Analysts work in the unified M365 Defender portal where both XDR and Sentinel incidents appear in a single queue. Azure Portal Sentinel is for administration only.
