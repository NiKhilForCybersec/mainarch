SOC Workflows

Standard operating procedures for incident triage, investigation, and response in the unified M365 Defender portal (security.microsoft.com). XDR and Sentinel incidents appear in a single queue.

Unified SOC Experience

Primary Interface: security.microsoft.com — XDR incidents (endpoint, identity, email) and Sentinel incidents (network, Azure, DLP) appear in the same queue.

XDR Benefit: Endpoint/identity/email incidents arrive pre-correlated with attack story, timeline, and MITRE mapping. Analysts get context immediately.

Common Investigation Queries

KQL - User Activity Timeline
// Get all activity for a user in last 24 hours
let targetUser = "user@revflow.com";
union SigninLogs, AuditLogs, DeviceLogonEvents
| where TimeGenerated > ago(24h)
| where UserPrincipalName =~ targetUser 
    or TargetUserName =~ targetUser
    or AccountUpn =~ targetUser
| project TimeGenerated, 
    Source = $table, 
    Activity = coalesce(OperationName, ActionType),
    Details = coalesce(ResultDescription, AdditionalDetails)
| order by TimeGenerated desc
KQL - IP Reputation Check
// Check if IP appears in threat intel
let suspiciousIP = "1.2.3.4";
ThreatIntelligenceIndicator
| where NetworkIP == suspiciousIP or NetworkSourceIP == suspiciousIP
| project TimeGenerated, ThreatType, Description, ConfidenceScore

Industry Best Practices

SOC Operations Best Practices

SANS SOC Recommendations:

  • Tiered Response: L1 triage (15 min), L2 investigate (1 hr), L3 hunt/respond
  • Playbooks: Documented procedures for common alert types
  • Metrics: Track MTTD, MTTR, alert volume, false positive rate
  • Continuous Improvement: Post-incident reviews, detection tuning

NIST Incident Response (800-61):

  • Preparation → Detection → Containment → Eradication → Recovery → Lessons Learned
  • Document all incidents for compliance and improvement
  • Establish communication channels for escalation
