The Consultant Mental Model

This framework applies to every company, every engagement. Different business, different assets, same thinking. Experienced security professionals sound confident not because they memorized tools, but because the structure never changes.

Remember This Forever

Business ‚Üí Planes ‚Üí Assets ‚Üí Risks ‚Üí Controls ‚Üí Telemetry ‚Üí Detections

That's it. Everything fits into this sequence.

1Ô∏è‚É£ Business (WHY the company exists)

This is ALWAYS the starting point. Ask yourself:

  • What does this company sell?
  • How does it make money?
  • What would hurt the business most if compromised?
QuestionRevFlow Analytics Answer
What does the company sell?Revenue intelligence, deal prediction analytics
How does it make money?B2B SaaS subscriptions from enterprise clients
What does it depend on?Salesforce integration, meetings data, customer CRM data
Biggest fears?Data leakage, account takeover, customer trust loss
Memory Hook

Security exists to protect how the company makes money.

If you skip this step, everything else is wrong.

2Ô∏è‚É£ Planes (HOW the company is structured)

This is where most people get confused ‚Äî but it's actually simple. There are 3 planes in every company:

üü¶ Control Plane
Who can do what
  • Identity
  • Access
  • Permissions
  • Cloud management

Examples: Entra ID, RBAC, MFA, Azure Activity Logs

üü© Data Plane
Where business actually runs
  • Endpoints
  • Applications
  • SaaS
  • Databases
  • Network traffic

Examples: MDE, Salesforce, App logs, Network flows

üü® Operations Plane
How things change
  • CI/CD
  • Automation
  • Admin actions
  • SOC workflows

Examples: GitHub, Azure DevOps, Service principals, Change logs

Why Planes Matter

Control Plane: If someone abuses identity, they control everything.

Data Plane: This is where value and damage happen.

Operations Plane: Attackers love abusing automation.

Memory Trick
Control = ACCESS
Data    = BUSINESS  
Ops     = CHANGE

Critical Distinction (Interview Gold)

MDI detects ATTACKS (Pass-the-Hash, Kerberoasting)

Entra ID Logs record CHANGES (user created, role assigned, CA policy modified)

You need BOTH. MDI won't tell you who was added to Global Admins ‚Äî that's in AuditLogs.

Risk comes before tools. You identify what can go wrong BEFORE you pick what to deploy.

6Ô∏è‚É£ Telemetry (WHAT you log)

Controls generate data. You ask: What signals prove misuse? Where does evidence come from?

Control AreaTelemetry SourceWhat It Shows

Critical Point

No telemetry = blind security. You can have the best tools, but without logs flowing to your SIEM, you can't detect anything.

7Ô∏è‚É£ Detections (HOW you catch bad behavior)

Finally: Correlate signals, detect abuse, alert SOC.

Detection Use CaseSignals CorrelatedWhat It Catches
Account Compromise Risky sign-in + endpoint anomaly Attacker using stolen credentials
Privilege Escalation Admin role change + unusual API calls Insider threat or compromised admin
Data Exfiltration Data download + external sharing Data theft attempt
Lateral Movement Multiple failed logins + successful login on different host Attacker moving through network
Key Insight

Detections are the final output ‚Äî not the starting point.

If someone asks "what detections do you have?" and you start listing rules, you're thinking backwards. Start with business, work forward.

Interview Application

When asked "Tell me about a security architecture you designed" or "How would you approach securing X company?":

Framework Response
"I always start by understanding the business ‚Äî what they sell, how they 
make money, and what would hurt them most. Then I map the environment 
into three planes: control (identity/access), data (where business runs), 
and operations (how things change).

From there, I identify critical assets in each plane, assess the realistic 
risks to those assets, and then select controls to mitigate those specific 
risks. The controls generate telemetry, which flows to the SIEM, where we 
build detections that correlate signals across planes.

For RevFlow, a B2B SaaS company, the biggest risks were account takeover 
and data exfiltration. So we prioritized identity controls (MFA, CA, PIM), 
endpoint protection (MDE with ASR), and data loss prevention (Purview DLP), 
with all telemetry flowing to a centralized Sentinel workspace for 
correlation and detection."
