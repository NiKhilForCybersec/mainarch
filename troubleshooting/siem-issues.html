Sentinel Issues

Common Microsoft Sentinel problems and their solutions.

Connector Not Ingesting Data

SymptomPossible CauseSolution
Table empty after connector enabledPermissions issueVerify connector has required Graph/Azure permissions
Data delayed 30+ minutesSource lag or workspace throttlingCheck source system, review workspace health
Partial data (some events missing)DCR filtering too aggressiveReview DCR XPath queries, broaden if needed
AWS connector failsSQS permissions or queue emptyVerify IAM role, check S3 notifications to SQS

Check Ingestion Health

KQL
// Check if data is flowing for each table
union withsource=TableName *
| where TimeGenerated > ago(1h)
| summarize EventCount = count(), LastEvent = max(TimeGenerated) by TableName
| order by LastEvent asc

High Ingestion Costs

SymptomPossible CauseSolution
Unexpected cost spikeNew log source or misconfigurationCheck Usage table, identify source, add filtering
Firewall logs too expensiveLogging all trafficMove to Basic Logs tier, filter informational
NSG flow logs expensiveHigh network activityUse Basic Logs tier, aggregate or sample

Identify Cost Drivers

KQL
Usage
| where TimeGenerated > ago(7d)
| where IsBillable == true
| summarize TotalGB = sum(Quantity)/1000 by DataType
| order by TotalGB desc
| take 10
