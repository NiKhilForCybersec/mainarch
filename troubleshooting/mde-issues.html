MDE Issues

Common Microsoft Defender for Endpoint problems and solutions.

Onboarding Failures

SymptomPossible CauseSolution
Sense service not startingMissing license or prerequisitesVerify M365 E5/E3+MDE P1/P2 license assigned
Device not appearing in portalNetwork connectivityCheck *.securitycenter.windows.com access
Onboarding script failsScript already ran or permissionsCheck OnboardingState registry, run as admin
Intune policy not applyingDevice not Intune enrolledVerify Entra ID join and Intune enrollment

Verify Onboarding Status

PowerShell
# Check onboarding state
$reg = Get-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Advanced Threat Protection\Status" -ErrorAction SilentlyContinue
if ($reg.OnboardingState -eq 1) {
    Write-Host "Device is onboarded" -ForegroundColor Green
} else {
    Write-Host "Device NOT onboarded" -ForegroundColor Red
}

# Check Sense service
Get-Service -Name Sense | Format-Table Name, Status, StartType

# Test connectivity
& "C:\Program Files\Windows Defender Advanced Threat Protection\MsSense.exe" -t

ASR Rule Conflicts

SymptomPossible CauseSolution
Application blocked unexpectedlyASR rule too aggressiveAdd exclusion for legitimate app path
Office macros blocked"Block macros" rule enabledExclude trusted macro locations
Scripts blocked from runningScript blocking rulesAudit mode first, then add exclusions

Check ASR Events

KQL
// ASR blocks in last 24 hours
DeviceEvents
| where Timestamp > ago(24h)
| where ActionType startswith "Asr"
| summarize BlockCount = count() by ActionType, FileName, FolderPath
| order by BlockCount desc
